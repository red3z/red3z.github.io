程序的栈和堆

程序的栈存放什么?

​ 每个线程对应一个栈, 栈里面是方法(函数)

​ 栈内存是每个线程私有的, 用于存储局部变量和方法调用的信息。‌

栈的空间怎么释放?

​ 栈的空间是自动释放的, 原因是方法一结束, 方法自动弹出栈

堆存放什么?

​ new出来的对象存放在堆中, 垃圾回收主要回收堆内存的对象

世界上最难调试的bug:

  • 野指针(NullPonterException)
    • 同一个对象, 两个指针, 一个释放了, 另一个不知道还在用
    • 不再指向任何对象的指针就是野指针
  • 并发问题
    • 多线程访问同一块内存空间

程序中的垃圾

什么是垃圾?

​ 没有引用指向的内存地址(或者说对象), 就是垃圾

怎么找到垃圾?

方法一: Reference Count引用计数法

​ Python使用

​ 每个引用都有一个计数器记录引用他的数量, 数量为0的就是垃圾.

​ 缺陷: 循环引用的垃圾找不出来.

方法二: ==Java使用== 根可达算法

​ 什么是根? main方法里建立的对象

GC算法

GC算法用来干什么?

​ GC算法GC(Garbage Collection) Algorithms指垃圾清理方式

GC算法有哪些?

  • Mark-Sweep(标记清除): 清除后内存容易碎片化
  • Copying(拷贝): 设定一些内存不允许使用, 根可达找到的对象全放到不可用内存中, 然后将可用内存全部清除. 浪费内存
  • Mark-Compact(标记压缩): 清理后就整理好. 效率低

垃圾回收器GC

为什么有那么多垃圾回收器?

​ Garbage Collector随着内存的增长演化

JVM的垃圾回收器怎么设计的?

​ JVM的垃圾回收机制基于GC分代算法设计，‌没有经历过垃圾回收器扫描的对象为新生代, 经历过的对象叫老年代, 90%的对象一次就被回收

JVM的垃圾回收器怎么分类的?

  • 单线程STW(Stop-The-World):

    • Serial: Copying, 年轻代

    • Serial Old: Mark-Sweep-Compact, 老年代

  • 多线程STW:

    • Parallel Scavenge, 年轻代
    • Parallel Old, 老年代
    • ParNew(Parallel New 工作在年轻代的多线程): 对比Parallel Scanvenge有些增强可以与CMS更好的配合使用

  • Concurrent: GC线程和业务线程同时

    • CMS(Concurrent Mark Sweep)
      1. 初始标记: STW找到根
      2. 并发标记: Concurrent执行业务同时标记垃圾
        • 会出现标记错的情况例如:垃圾在运行过程中又有个引用指向他, 又不是垃圾了(缓存命中, 没命中的要清除了, 突然又来了个命中了; 重写了finalize(),死了的对象又给复活了); 本来不是垃圾, 运行过程中引用没有了, 又变成垃圾了;
      3. 重新标记: STW, 修正标错的垃圾(必须从头扫描, 但是第二阶段并发标记仍然有用, 因为梳理了相关的内容, 简化了一些东西)
      4. 并发清除

CMS的三色标记算法

三色标记算法用在什么时候?

​ 三色标记用在并发标记阶段

三色用来干什么?

​ GC线程受操作系统调度隔一段时间运行一段, 所以需要保存运行状态, 用三色标记算法来记录状态

三色是什么?

  • 黑色: 自己标记了, 孩子(Fields)也标记了; 不再扫描
  • 灰色: 自己标记了, 孩子(Fields)还没; 扫描孩子
  • 白色: 没有标记到的结点;

三色标记算法的注意事项:

​ 当某个黑色对象的孩子指向新的对象, 黑色对象变为灰色

三色标记算法无法解决的Bug

​ 多个GC线程存在无法解决的Bug

G1

G1怎么解决三色标记算法并发的问题的?

​ 三色标记算法 + SATB(Snapshot At the Begining)来解决并发存在的问题, 当业务线程执行了引用的改变时, 这个修改记录会被记录下来, GC线程根据记录来继续下一步的垃圾回收.

​

G1为什么效率更高?

​ G1屏弃了分代思想, 物理上不分代, 逻辑上分代

​ G1算法将内存等分成块(分区), 叫作region.

分区的好处是什么?

​ 老年代满的时候, 不需要回收一大块, 可以选择回收一些区域

​ 业务逻辑不用停

​ 每个region可以在逻辑动态改变新生代还是老年代

JVM调优

什么是调优?

  1. 根据需求进行JVM规划
    • 用什么机器, 用多少内存
  2. JVM运行慢, 卡顿
    • 定位系统的瓶颈, 使用压力测试
  3. 运行过程中的各种问题
    • 比如Memory Leak, OOM

常用命令:

  • jps(JVM Process Status)
  • jinfo
  • jstate(JVM Statistics)
  • jstack
  • top查看CPU占用情况
    • top -Hp ThreadID查看线程的CPU占用情况
  • jmap -histo ThreadID查堆内存不同类对象占用的比重
    • jmap -dump:format=b, file=20240801.hprof将整个堆导成一个文件

如何查JVM问题?

生产环境不能直接导文件, 因为会使程序卡死

  1. 在压力测试环境里图形化远程观察
  2. 负载均衡, 把其中一台摘出来把堆转储文件导出来
  3. tcp dump复制连接请求一份, 一份打到生产环境, 一份打到测试环境里

JVM

JVM充当翻译官, 把.class字节码文件(.java文件编译后)翻译成当前平台认识的可执行文件的格式并执行

JVM在JDK(Java Development Kit)里面

JRE(Java Runtime Environment), Java的运行环境

JIT是什么东西?

Just-In-Time compiler, JIT编译器把.class文件编译成本地代码(本地代码可以在当前系统直接运行, 而不需要在JVM上运行),

JRE是什么?

Java Runtime Environment Java运行时环境

JRE = JVM + core lib, core lib里面包含String.class, Object.class等核心类

JDK是什么?

Java Development Kit

JDK = JRE + development kit

Java一个指针占多少字节?

​ 64位虚拟机默认开启压缩, 一个指针是4个B. 关闭压缩则是8个B

一个对象在堆内存中, 都有哪些东西

  • Markword大小8B
  • Class pointer默认大小4B, 关闭压缩8B
  • Instance data
  • Padding保证对象大小能被8B整除

Markword记录什么东西?

  • 记录垃圾回收器的信息: 三色的颜色信息, 分代年龄
  • 记录锁信息
  • 记录Hashcode信息

对象怎么定位? T t = new T();t是怎么找到对象的

​ 直接指针: t指向堆中new出来的空间, 这个空间内有Classpointer指向方法区中的T.class

​ 间接指针: GC回收较快

对象怎么分配?

​ 对其进行分析, 如果可以的话在栈上分配;

​ 判断这个对象大不大, 大的话放在老年代;

​ TLAB(Thread Local Application Buffer): 防止好多线程往伊甸园区放对象, 出现锁竞争影响效率. TLAB为每个线程分配一个私人空间

​ 经过垃圾回收一次后, 去往Survival区

往栈上分配对象的标准是什么?

逃逸分析: 如果这个对象只在方法局部使用, 方法结束可以随着方法一起弹出栈

对象的弱引用是什么?

垃圾回收器看到弱引用就当垃圾看,

强引用

软引用: 内存不够的话回收软引用, 给重量级角色让位, 用作缓存

弱引用: 通知垃圾回收器进行特殊处理

虚引用: 在ThreadLocal里防止内存泄露

JVM内存结构内存模型

常量池有哪些?

  • Run-Time Constant Pool运行时常量池

    • 运行时常量池存放 ==字面量== 和 ==符号引用==, 如果一个 'string'还不是String对象, 当Java程序运行执行到这个 'string'字面量时, 会先去字符串常量池找该字面量的对象引用是否存在, 存在则返回; 不存在则在堆创建该字面量的String对象, 并将其引用放在字符串常量池中, 再返回
    • 'string' JVM启动后 'string' 已经进入运行时常量池
    • JVM加载class时, 会将class文件常量池导入在运行时常量池, 多个class文件常量池中的相同字符串只会对应运行时常量池中的一个字符串
    • 保存符号引用对应的直接引用(实际内存地址)。这些直接引用是JVM在类加载之后的链接(验证、准备、解析)阶段从符号引用翻译过来的。
    • 运行时常量池具有动态性的特征，它的内容并不是全部来源与编译后的class文件，在运行时也可以通过代码生成常量并放入运行时常量池。比如String.intern()方法。

  • class文件常量池

    • 每个class字节码文件都有一个常量池, 存放编译后就知道的, 该class会用到的 ==字面量== 和 ==符号引用==. JVM加载class, 将其类信息, class文件常量池置于方法区.

      • 字面量: 文本字符串, final修饰的成员变量的值,

      • 符号引用: 类和接口的完全限定名, 方法的名称和描述符, 字段的名称和描述符, 局部变量

    • class类信息及其class文件常量池是字节码的二进制流, JVM加载类时, 将其转化为 Class类的对象实例; 将class文件常量池中的内容导入 ==运行时常量池==

  • 字符串常量池

    • 是JVM维护字符串实例的 ==引用表==.存String对象的引用. 在HotSpot中被实现为一个全局的StringTable, 底层是C++的hashtable, 字面量作为key, 实际堆中创建的String对象的引用作为key.
    • 字符串常量池在逻辑上属于方法区，但JDK1.7开始，物理上就被挪到了堆区。

  • 基本数据包装类常量池

    • Float和Double没有常量池
    • Byte,Short,Integer,Long : [-128~127]
    • Character : [0~127]
    • Boolean : [True, False]

JVM内存模型

  • ==JVM Stacks==栈
  • ==Heap==堆
  • ==PC Register==程序计数器: 记录每个线程执行到哪了
  • ==Method Area==方法区(方法区是抽象的规范, 方法区只是一个概念, 元空间和永久代是对抽象的具体实现): 存放常量池, 类加载到的Class对象, 静态变量
    • Run-Time Constant Pool运行时常量池
    • class文件常量池
    • 字符串常量池
    • 基本数据包装类常量池
  • ==Native Method Stacks==本地方法栈: 本地方法就是native关键字修饰的方法

JVM哪些区域是线程共享的?

​ 堆和方法区

栈帧是什么?

栈帧是栈中的一个栈元素, 栈帧包含了方法的信息

类加载机制

类加载是干啥?

用编译过的二进制class文件, 生成某种形式的Class数据结构进入内存, 程序员调用这个数据结构去构造对象

什么时候会类加载?

类加载机制是在程序运行时进行的

  • 类的实例化, 子类的实例化
  • 访问静态==成员==
  • 反射加载类
  • JVM启动时加载的类

类加载的过程

  1. Loading加载: 将class文件(二进制)转化为某种数据结构存储在==方法区==, 在堆中生成一个便于用户调用的Java.lang.Class类型的对象
  2. Linking连接(JVM掌控)
    • 验证Verification: 验证该class是否符合JVM标准
    • 准备Preparation: 静态变量==赋默认值==
    • 解析Resolution: 将类, 方法, 属性等符号引用替换为直接引用(内存地址), 常量池中的各种符号引用解析为指针, 偏移量等内存地址的直接引用
  3. Initializing初始化: 调用类初始化代码, 成员变量的赋初值, 静态变量==赋初始值==, 调用静态代码块. (只有调用new指令才会调用构造函数)

对象创建过程

  1. 申请空间, 申请来的空间内都是默认值(半初始化)
  2. 调用构造方法, 赋初始值
  3. 建立关联(将变量指向内存中的地址)

类加载器

每个类加载器加载一类class文件

  • BootStrap加载lib/rt.jar charset.jar, C++实现
  • Extention加载jre/lib/ext/*.jar
  • Appication加载classpath内容
  • CustomClassLoader自定义

误区: 与父加载器不是继承关系, 父加载器也不是类加载器的加载器

Class对象在堆还是在方法区?

对象的Markword存放方法区的C++对象地址, 这个C++对象包含了指向存在堆中的Class对象

双亲委派的过程

类加载器收到类加载请求时，先请求父类加载器去加载。当父类加载器无法加载该类时，子类加载器才会加载。

双亲说的是子到父父再到子

为什么要双亲委派?

  1. 为了安全, 如果没有的话, 自己可以写String类
  2. 避免类的重复加载
